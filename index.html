The Vulnerability

Computers Trust Blindly.

When you plug in a USB keyboard, your computer accepts it immediately. It does not check who is typing.

Hackers use tools like the "USB Rubber Ducky." It looks like a flash drive but acts like a keyboard. Once plugged in, it types malicious commands in seconds‚Äîfaster than any human can react.

Why Software Fails:

  • ‚úï Antivirus thinks it's just you typing.
  • ‚úï Firewalls don't block keyboards.
The Solution

Physical Verification.

We propose a hardware device that sits between the USB gadget and the computer. It physically cuts the wires used for data.

The computer sees nothing until a human user types a random 4-digit PIN displayed on the device. This proves a human is present.

How it Protects:

  • ‚úì Air Gap: Wires are physically disconnected.
  • ‚úì Human Check: Automated scripts can't read the PIN.

// Main Loop
void loop() {
  // 1. Cut the connection initially
  disable_usb_output();

  // 2. Wait for a keyboard to be plugged in
  if (keyboard_detected()) {
    
    // 3. Generate random code
    int code = random(1000, 9999);
    screen.print(code);

    // 4. Wait for user to type code
    if (user_types_correct_code(code)) {
       enable_usb_output(); // Connect wires!
    }
  }
}
